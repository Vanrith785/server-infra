## Issues Found and Resolved

1. **Ghost containers** — NPM and all backend services were running in
   containerd but invisible to Docker (`docker ps`). Caused by Docker
   daemon state loss, likely from prior daemon restart without live-restore.
   Fixed by removing stale state files from /var/lib/docker/containers/
   and redeploying via docker compose.

2. **Stale iptables DNAT rules** — Old ghost NPM (IP 192.168.128.2) had
   three DNAT rules for ports 80/81/443 that persisted after ghost cleanup,
   hijacking all traffic before the new NPM rules. Deleted manually.

3. **Snap Docker bind mount restriction** — Docker (snap v29.3.1) silently
   creates empty tmpfs mounts for paths outside /root/ or /home/. NPM data
   at /data/compose/1/ was inaccessible. Migrated to /root/npm-data and
   /root/npm-letsencrypt.

4. **Docker subnet pool exhaustion** — 16 /20 subnets already allocated.
   New services (attendance, diabetes, policy) configured to use the
   existing 'production' network instead of creating new ones.

5. **NPM network isolation** — Freshly redeployed NPM was on an isolated
   network, unable to reach any backend by container name. Connected NPM
   to all 5 relevant networks.

6. **vantapolicy SSL missing** — nginx config for vantapolicy only had
   listen 80, no listen 443 or ssl_certificate directives. Manually patched
   and cert#31 registered in NPM database.

7. **Portainer network isolation** — Portainer ghost cleared and redeployed
   with --network production so NPM can reach it by container name.
